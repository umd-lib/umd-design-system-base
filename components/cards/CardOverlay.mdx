### Implementation Note

Standard cards and overlay cards differ primarily in image handling. The standard card should always include an image, while the overlay card should not. To maintain this consistancy, those will be implemented as separate components.

### Twig

#### Overlay Card

If possible, nested components(secondary button for example) should be used to build the card. This allows for easier maintenance and updates.

```twig
{# Card Overlay Component #}
<div role="contentinfo" class="card--overlay c-bg-secondary c-content-primary">
 <div class="card--content s-box-small-v s-box-small-h">
   <div class="card--title">
     {% if eyebrow %}
       <div class="card--eyebrow t-eyebrow s-stack-small">
         <p>{{ eyebrow }}</p>
       </div>
     {% endif %}

     <h3 class="card--headline s-stack-default t-title-medium">
       <a href="{{ url }}">{{ title }}</a>
     </h3>
   </div>

   <div class="card--details">
     {% if description %}
       <div class="card--text t-body-small c-content-secondary s-stack-default">
         <p>{{ description }}</p>
       </div>
     {% endif %}

     {% if date %}
       <div class="card--date t-label c-content-tertiary s-stack-large">
         <time datetime="{{ date|date('Y-m-d') }}">{{ date|date('F d, Y') }}</time>
       </div>
     {% endif %}
   </div>

   {% if cta %}
     <a href="{{ cta.url }}" class="button--secondary t-body-small t-interactive-sub c-content-primary c-underline-primary ani-underline l-flex-row">
       <div class="i-chevron"></div>
       {{ cta.text }}
     </a>
   {% endif %}
 </div>
</div>
```
